# ロック画面マジックアプリ 製品仕様書 v1.0

**アプリ名（内部名）**: lockscreen
**配置先**: `apps/lockscreen/lockscreen.html`（1ファイル完結）
**ホスティング**: `https://finaloji.github.io/nandeya-studio/apps/lockscreen/lockscreen.html`

---

## マジックの全体像

「失敗→的中型」の単独演目。開錠そのものがクライマックス。

| 誰が | 何をするか |
|------|-----------|
| マジシャン | 隠しジェスチャーで演出をONにしてから端末を観客に渡す |
| 観客 | 好きなパスコードを入力する → 何を打っても「間違い」（シェイク演出） |
| 観客/マジシャン | 緊張感が高まったところで再度入力 |
| 端末 | 設定時間（デフォルト30秒）経過後は、どんな数字でも開錠 → ホーム画面が表示される |

- 数字のフォースは行わない。「ロックされていたのに開いた」という不可能性が効果の核
- 開錠後はホーム画面（ダミーアイコンの並ぶ画面）が見えて演目終了

## 画面構成

### 1. ロック画面（パスコード入力）

iOSの実機パスコード画面に忠実に寄せる。
※実機のパスコード入力画面には時計・日付は表示されないため、このアプリでも表示しない。

- 上部: 「パスコードを入力」の文言
- その下: 入力ドット（桁数ぶんの○。入力で塗りつぶし）
- テンキー: 1〜9・0 の円形ボタン（2〜9にはABC等の英字添え字）
- 下部: 「緊急」（左）「キャンセル」（右）のテキストラベル
- 背景: 暗めの壁紙にすりガラス風のぼかし

### 2. ホーム画面（開錠後）

- 壁紙＋ダミーアプリアイコンのグリッド＋下部Dock
- 触っても何も起きない静的な画面でよい（クライマックスの数秒間見えれば十分）

---

## パスコード入力の挙動

- タップで左からドットが埋まる
- 全桁入力された時点で自動判定（iOSと同じく決定ボタンなし）
- **不正解時**: ドット列が左右にシェイク → 全ドットが空に戻る（バイブ相当の演出）
- **正解（開錠）時**: 開錠アニメーション → ホーム画面へ遷移

### 判定ロジック

| 状態 | 判定 |
|------|------|
| 演出OFF | 常に不正解（何を打っても開かない） |
| 演出ON・開始から設定時間内 | 常に不正解 |
| 演出ON・設定時間経過後 | 常に正解（どんな数字でも開錠） |

- 開錠したら演出は自動的にOFFに戻る

---

## 隠し操作

### 演出ON/OFF（機能起動コマンド）

**ロック画面上の特定要素を3秒長押し**（候補: 「緊急」ラベル。スプリント2-1で確定） → 演出ON（タイマー開始）/ 再度3秒長押しでOFF

- 端末を渡す直前に自然に仕込める
- ONサイン（術者だけが分かる変化）を1箇所だけ用意する（例: 文言のフォントウェイトが僅かに変わる等。詳細はスプリントで決定）

### 入力履歴の確認

**画面を2本指で上にスワイプ** → 画面下部に入力履歴パネルを表示

- これまでの全試行を時系列リストで表示（試行番号＋入力された数字）
- パネル外タップ or 2本指下スワイプで閉じる
- 履歴は演出ONのタイミングでリセット

### 隠し設定

設定できる項目:

| 項目 | 選択肢 | デフォルト |
|------|--------|-----------|
| パスコード桁数 | 4桁 / 6桁 | 6桁 |
| ロック解除までの時間 | 秒数指定 | 30秒 |

- 設定への入り方も隠し操作にする（詳細はスプリントで決定）
- 設定は端末に保存され、再読み込み後も維持される

---

## データの保存

- 桁数・ロック時間の設定は保存する
- 演出ON状態と開始時刻も保存する（演技中にページが再読み込みされても30秒カウントが壊れないように）
- 入力履歴はセッション中のみ保持でよい

---

## 難読化方針（最重要）

stopwatch / calc と同じ方針。コードを見ても「時間経過で必ず開く」仕掛けが分からないようにする。

- localStorage キーは意味のない名前（`_lk1` 系など）
- 関数名・変数名も難読化
- コメントでは「表示調整」「入力補正」などの言葉を使い、マジック用語を書かない
- 具体的な対応表は実装スプリントで確定し、この文書に追記する

---

## スマホ最適化

- `touchstart` / `touchend` でボタン応答（click遅延を避ける）
- `user-select: none`、`-webkit-tap-highlight-color: transparent`、`touch-action: manipulation`
- viewport: `width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no`
- PWA対応: manifest.json、apple-touch-icon、ステータスバー黒
- 縦向き前提
