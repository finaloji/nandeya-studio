# 電卓マジックアプリ 製品仕様書 v3.0

**アプリ名（内部名）**: calc-force  
**配置先**: `apps/calc/calc.html`（1ファイル完結）  
**ホスティング**: `https://finaloji.github.io/nandeya-studio/apps/calc/calc.html`

---

## マジックの全体像

全員が自分のスマホの電卓を使って同じ計算をする。

| 誰が | 何をするか |
|------|-----------|
| 観客全員 | 自分のスマホの電卓アプリを開く |
| マジシャン | この特製電卓アプリを使う |
| 全員 | 同じ数字を順番に足し算していく |
| マジシャン | ＋を3秒長押し → アプリが「帳尻の数字」を自動計算して画面に表示 |
| マジシャン | 「この数字を入力してください」と画面の数字を読み上げる |
| 観客全員 | その数字を自分のスマホに打ち込んで＝を押す |
| 全員の画面 | 同じ結果（今日の日付＋時刻）が表示される |

---

## 表示する値のフォーマット

**`YYMMDDHHММ`（10桁）**

例：2026年7月1日14時47分 → `2607011447`

---

## フォース機能の仕組み

### 起動と計算

1. ＋を3秒長押し → フォース起動
2. 起動時点の「1分後の日時」を目標値として自動計算
3. **目標値 − 現在の累積合計 ＝ 帳尻の数字** を算出
4. この帳尻の数字をディスプレイに表示（10桁・ゼロ埋め）
5. マジシャンが読み上げ → 全員が入力 → 全員＝を押す → 全員の画面に同じ日時

### 具体例

| 項目 | 値 |
|------|-----|
| 起動時刻 | 14:46:30 |
| 目標値（1分後） | `2607011447` |
| 観客の合計（406+178+5698+3265） | `9547` |
| 帳尻の数字（画面に表示） | `2607001900` |
| 全員が入力して＝を押すと | `2607011447` |

### 累積合計の正確な取得

フォース起動時（＋長押し）のタイミングに応じて以下を使う：

| 状態 | 累積合計の取得方法 |
|------|-----------------|
| 演算子押下後・次の入力前 | `_op1`（蓄積済み） |
| 演算子押下後・入力中 | `_applyOp(_op1, _opc, parseFloat(_cur))` |
| 演算子なし | `parseFloat(_cur)` |

### エラー処理

- 帳尻の数字が負（合計 > 目標値）の場合：フォース起動せず、ACを3回点滅して通知

## 電卓の通常動作

- **数字入力**: 最大9桁・小数点1回のみ
- **四則演算**: チェーン計算対応
- **＝連打**: 直前の演算子と第2オペランドを繰り返し適用
- **⌫（バックスペース）**: 入力中は末尾1桁削除。計算後は「0」にリセット
- **AC / C**: 数字入力中は「C」（入力クリア）・それ以外は「AC」（全リセット）
- **+/−**: 符号反転
- **%**: 現在値を100で割る（演算子押下後はiOS準拠）
- **0÷0**: エラー表示

---

## 隠し操作

**＋ボタンを3秒長押し** → フォース起動 / 再度3秒長押しでキャンセル

- 演技の流れで「次の方に足し算させる」タイミングに自然に溶け込む
- 長押し中は通常の＋動作をしない（3秒経過でフォース発動）

---

## ONサイン（術者だけが分かるサイン）

**ACボタンのフォントウェイトのみ変化**

| 状態 | ACのフォント |
|------|------------|
| OFF | `font-weight: 300`（細め） |
| ON | `font-weight: 700`（太め） |

他のボタン・ディスプレイは一切変化しない。
