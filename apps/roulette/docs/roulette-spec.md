## マジックの全体像

普通の「意思決定ルーレット」に偽装した単独演目。何気なく取り出し、観客に止めさせても、術者が仕込んだ項目（例：ドンペリ）に必ず止まる。

| 誰が | 何をするか |
|------|-----------|
| マジシャン | 事前に隠し設定で「項目リスト」と「当たり（フォース先）」を仕込んでおく |
| マジシャン | 「おかわりをルーレットで決めましょう」とアプリを取り出して円盤を回す |
| 観客 | 好きなタイミングで画面をタップして止める |
| 円盤 | どのタイミングで止めても自然に減速し、必ず当たり（ドンペリ）で止まる |
| 全員 | 「ドンペリだ！」で盛り上がって演目終了 |

- 数字のフォースはしない。「自分で止めたのに狙った項目になった」という不可能性が核。
- 見た目・触り心地は市販の抽選ルーレットと区別がつかないようにする（偽装が命）。

## ルーレット形式（確定）

**回転する円盤（ホイール）＋上部に固定の矢印**

- 円盤は項目数ぶんの扇形（セグメント）に等分。各セグメントにラベル（ドンペリ／ビール／…）。**初期は6セグメント**。
- 矢印は画面上部（12時方向）に固定。円盤が回転して減速し、矢印が指すセグメントが結果。
- セグメントの色は選択中のパレット（後述）から順に割り当てる（隣接色がかぶらないよう配色）。
- **描画は SVG**。セグメントは `<path>`、ラベルは `<text>`（各セグメントの中心角に沿って配置）。回転は円盤グループ全体を `transform: rotate()` で回す。テキストが鮮明で、画面幅に応じた可変にも強い。
- **ラベルは自動縮小**：セグメント幅に収まらない長さのときはフォントサイズを段階的に下げて1行に収める（省略も折返しもしない）。極端に長い場合の下限フォントは決めておく。

### 画面構成

```
┌─────────────────┐
│      ルーレット      │ ← タイトル（隠し設定を開くタップ領域）
│        ▼          │ ← 固定の矢印（12時／アームのON/OFFサインを兼ねる）
│    ╱───────╲      │
│   │ ドンペリ │     │
│   │ビール 円盤 ハイ│ │ ← 回転する円盤（SVG・6セグメント＋ラベル）
│   │  ワイン  │     │
│    ╲───────╱      │
│                   │
│    ┌─────────┐    │
│    │  START   │    │ ← START/STOP ボタン（トグル）
│    └─────────┘    │
└─────────────────┘
```

- タイトル文字（表示名）を上部に小さく置く（隠し設定を開くタップ領域に使う。後述）。
- 中央下に **START/STOP ボタン**（1つのボタンがトグル）。アイドル時は「START」、回転中は「STOP」表示。

## 演出の流れ

1. アイドル状態：円盤停止。ボタンは「START」。
2. START を押す → 円盤が一定の高速で回り続ける（減速はまだしない）。ボタンは「STOP」に変化。
3. STOP を押す（観客に押させてもよい）→ その瞬間から**自然に減速して当たりに着地**。
4. 停止：矢印が当たりセグメントを指す。少し余韻を持たせて結果を強調。ボタンは「START」に戻る。
5. もう一度アイドルへ（次の演技に備えてリロード不要）。

## フォースの仕組み（このアプリの技術的な肝）

電卓フォースの「帳尻の数字」と同じ発想。**止めた瞬間の現在角度から逆算して、狙った角度にちょうど着地させる**。

### 円盤の角度モデル

- セグメント数 `N`、1セグメントの角度幅 `span = 360 / N`。
- 円盤の回転角 `_ang`（時計回りに増加、度）。矢印は常に12時固定。
- 当たりセグメント `_ti`（インデックス）を矢印下に持ってくる回転角を `targetAngle`（mod 360）とする。

### 減速着地の計算（STOP を押した瞬間）

```
θ0        = 現在の回転角（mod 360）
targetAngle = 当たりセグメントを矢印下に置く角度（mod 360）＋ ジッター
delta     = ((targetAngle − θ0) mod 360)      // 0〜359：あと少しで着く分
extraTurns = 2〜4（現在速度が速いほど多く）    // 自然に見せる追加回転
θ1        = θ0 + extraTurns*360 + delta        // 最終到達角
duration  = 2.5〜4.0 秒（extraTurns に応じて）
```

- `θ0 → θ1` を **ease-out 系（easeOutQuart 等、終盤ほど強く減速）** でアニメーション。
- 終盤に「ジリジリと当たりに吸い込まれる」動きがルーレットらしさを生む。ここが自然かどうかで成否が決まる。

### ジッター（毎回ど真ん中に止めない）

- `targetAngle` に `±(span/2 − マージン)` の範囲で微小なランダムオフセットを加える。
- 毎回セグメント中央ぴったりに止まると不自然・機械的なので、必ず当たり内に収まる範囲で揺らす。

### 早すぎる停止のガード

- 回し始め直後に STOP されても、`extraTurns ≥ 2`・`duration ≥ 2.5秒` を最低保証。
- カクッと即停止させない（バレる）。

### ランダムモード（フォースOFF時）

- `targetAngle` を当たり固定ではなく**ランダムなセグメント**にするだけ。減速アニメは完全に同一。
- OFF時は本当にランダムに止まる → 事前に「本当にランダムでしょ？」と見せられる（弁明可能性）。

## 回転の開始・停止（確定）

**中央下の START/STOP ボタンで回す・止める（1つのボタンがトグル）**

| 状態 | ボタン表示 | 押すと |
|------|-----------|--------|
| アイドル | START | 回転開始（高速で回り続ける） |
| 回転中 | STOP | 減速開始 → 当たりに着地 |
| 減速中 | STOP（無効） | 反応しない（着地まで待つ） |

- ボタンがあることで誤操作が少なく、術者が主導権を握りやすい。
- 観客に「好きなところで止めて」と STOP を押させてもよい。
- **補助操作**：回転中は円盤本体をタップしても停止できる（端末ごと観客に渡したいとき用）。減速開始後のタップ・連打は無視。
- START/STOP と円盤タップ以外の画面領域は回転に反応させない（隠し操作と競合させないため）。

## 演出ON/OFF（アーム）とONサイン（確定：アーム式）

弁明可能性のため、既定はOFF（本当にランダム）。クライマックス直前に術者が「アーム」してフォース状態にする。ロック画面アプリの演出ONと同じ思想。

| 状態 | 挙動 |
|------|------|
| OFF（既定） | STOP で止めると**ランダム**に着地（デモ用・弁明用） |
| ON（アーム済み） | STOP で止めると必ず**当たり**に着地 |

- **アーム操作**: 上部の**矢印（▼）を素早く3回タップ**でON/OFFトグル（アイドル時のみ有効・タップ間隔600ms以内）。中央は START/STOP ボタンが占めるため、矢印を仕込みポイントにする。
- **ONサイン**: ONの間だけ矢印の見た目がごくわずかに変化（色の濃さ or 影を1段階だけ）。術者だけが気づく程度に留める。矢印は常に視線の近くにあり確認しやすい。
- アーム状態は**セッションのみ保持**（リロードでOFFに戻る）。うっかりアームしたまま観客に渡ってもリロードで安全側に倒れる。
- 運用注意：観客が偶然、矢印を素早く3回タップするとOFFに戻る。クライマックス前にONサインを目視確認すること（ロック画面アプリと同じ注意）。

## 隠し設定（項目と当たりの仕込み）　※一部要相談

**タイトル文字を素早く3回タップ**（アイドル時のみ・タップ間隔600ms以内）→ 設定パネルを開く。

| 項目 | UI上の文言（難読化） | 内容 |
|------|--------------------|------|
| 項目リスト | 「表示項目」 | ルーレットのラベルを追加・編集・削除（例：ドンペリ／ビール／ハイボール…） |
| 当たり | 「基準項目」 | 上のリストから1つを選択＝フォース先。UIでは「当たり」「フォース」と書かない |
| 項目数 | （リスト連動） | 初期6。リスト編集に追従（上限は要検討） |
| 配色 | 「テーマ」 | 複数のプリセットパレットから選択（サムネイルで選ぶ）。選択で即プレビュー |
| 完了 | 「完了」 | 保存してパネルを閉じる |

### 配色パレット（プリセット）

- 隠し設定の「テーマ」で複数のプリセットから選ぶ。各パレットは4〜8色程度の配列。
- セグメントには先頭から順に色を割り当て、隣接が同色にならないようにする（項目数がパレット色数を超える場合は循環＋隣接回避）。
- 具体的な色パターンは実装スプリントで数種類用意（例：カジノ系／パステル系／モノトーン系など）。名称はネタバレしない中立的な呼び方にする。

- 設定は localStorage に保存し、再起動後も保持（演技途中でアプリを閉じても消えない）。
- パネルの開閉は回転中は無効（回転中のタップは「止める」専用のため）。
- 「モード」「フォース」「当たり」等ネタバレ語はUIに出さず「基準項目」等で統一（難読化ルール準拠）。

## ライセンス管理（確定：あり）

既存アプリ（stopwatch / calc）と同じ **ID・パスコード＋端末ロック方式**。詳細は `apps/stopwatch/docs/auth-spec.md` を踏襲する。ここでは差分のみ記載。

### 方式

- 起動時にログインゲートを表示。ID・パスコードで認証し、1IDにつき1端末のみ使用可（`deviceId` ロック）。
- Firestore（プロジェクト `nandeya-magic-app`）の `licenses` コレクションを共用。ドキュメントID＝発行ID。
- 端末変更は `deviceChangeRequests` に申請 → `admin.html` で承認して `deviceId` を null に戻す運用。
- ログイン済み記録は localStorage に保持し、再訪時は Firestore で端末IDの有効性を確認してから通常画面へ。

### 管理画面

- `apps/roulette/admin.html`（既存 admin と同型）。ID発行・一覧・削除・端末登録リセット・端末変更承認。
- 既存アプリと同じ `licenses` コレクションを共用する（いつもどおりの運用）。

### サブスク構想との整合（要相談）

- マジックアプリ群は月額サブスク（`members` コレクション＋Stripe＋Discord）への移行構想がある。ルーレットもそのプラットフォームに乗せる場合、認証は `licenses` 方式ではなく共通のサブスク会員判定に寄せる。
- **当面は既存の ID/パスコード方式で実装**し、プラットフォーム側が固まった段階で認証部分だけ差し替える方針とする（本体のフォース機能とは疎結合に保つ）。

## 難読化方針（最重要）

stopwatch / calc / lockscreen と同じ方針。コードを見ても「必ず当たりに止まる」仕掛けが分からないようにする。

- localStorage キー・関数名・変数名は意味のない名前。
- コメントは「補正」「基準」「表示調整」などの言葉。マジック用語（フォース・当たり・仕込み）を書かない。
- UIラベルも「表示項目」「基準項目」で統一。

## スマホ最適化

- `touchstart` / `touchend` でタップ応答（click遅延を避ける）。
- `requestAnimationFrame` で回転・減速をアニメーション。
- `user-select: none`、`-webkit-tap-highlight-color: transparent`、`touch-action: manipulation`。
- viewport: `width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no`。
- PWA対応: manifest.json、apple-touch-icon、ステータスバー黒。
- 縦向き前提。円盤は **SVG** で描画し、画面幅に応じて可変（`viewBox` で自動スケール、ラベルは常に鮮明）。
