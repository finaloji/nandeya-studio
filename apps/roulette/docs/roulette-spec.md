### v0.4での変更点

- ラベルの向きを「内側から外側に向けて読める」形に統一（位置による反転をやめ、全セグメントで同じ回転規則にする）。実装は `apps/roulette/roulette.html` に反映済み。
- **セグメント幅を項目ごとに個別調整できるようにする**（新機能）。「当たりの枠を極端に狭くしても必ずそこに止まる」という、より不可能性の高い演出を可能にするため。詳細は下記「セグメント幅の個別調整」章を参照。この変更は円盤の角度モデル・フォース計算の両方に影響するため、回転・停止（スプリント1-2）の実装前に仕様へ組み込む。

### v0.5での変更点（計画段階・未実装）

- **豪華演出（大当たり演出）を追加する**（新機能・コード未着手）。着地した項目が基準項目と一致したときに、パチンコの大当たり／確変演出のような派手な祝福演出を表示する。項目ごとに任意で写真を登録でき、登録済みならその写真も表示する。詳細は下記「豪華演出（大当たり演出）」章を参照。

### v0.6での変更点（実機フィードバックによる改善・計画段階）

1. **画面上部の白い帯を解消**：PWAとしてホーム画面から開いたとき、iPhoneのノッチ／ステータスバー領域が白いまま残る不具合。`viewport-fit=cover` の指定とセーフエリア対応CSSが抜けているのが原因と推定。背景の黒をノッチ下まで届かせる。
2. **隠し設定パネルの入口を、普通に見える「設定」ボタンに変更**：これまでの「タイトル文字を3回連続タップ」を廃止し、画面左上に常時表示のボタン（歯車アイコン等）を置く。設定パネルの中身は既存どおりネタバレしない語（表示項目／基準項目／テーマ／範囲）のままなので、入口を隠す必要はないという判断。
3. **演出モードON/OFF（アーム）のジェスチャーを簡略化**：隠しジェスチャー自体は維持しつつ、矢印3回連続タップより簡単な操作に変更する。有力案は**矢印の長押し（1回の操作）**。既存のON/OFF切替のロジック・ONサイン（矢印の微変化）は維持し、判定条件（連打→長押し）のみ変更する。
4. **基準項目以外の出目にも簡易演出を追加**：現状「大当たり演出（大）」しか無く、それ以外は静かな余韻のみで地味だった。演出を3段階にする。
   - **大**：既存の大当たり演出（FEVER）。基準項目専用、常にこの段階。
   - **中**：矢印まわりに小規模な紙吹雪＋短い音。
   - **小**：短い白フラッシュ（0.3秒程度）＋短い「ピロン」音のみ。全画面オーバーレイは使わない。
   - **中／小の割り当ては項目ごとに隠し設定で決められる**（表示項目の編集に「演出」の選択肢を追加するイメージ。基準項目は常に「大」固定で選択不可）。

## マジックの全体像

普通の「意思決定ルーレット」に偽装した単独演目。何気なく取り出し、観客に止めさせても、術者が仕込んだ項目（例：ドンペリ）に必ず止まる。

| 誰が | 何をするか |
|------|-----------|
| マジシャン | 事前に隠し設定で「項目リスト」と「当たり（フォース先）」を仕込んでおく |
| マジシャン | 「おかわりをルーレットで決めましょう」とアプリを取り出して円盤を回す |
| 観客 | 好きなタイミングで画面をタップして止める |
| 円盤 | どのタイミングで止めても自然に減速し、必ず当たり（ドンペリ）で止まる |
| 全員 | 「ドンペリだ！」で盛り上がって演目終了 |

- 数字のフォースはしない。「自分で止めたのに狙った項目になった」という不可能性が核。
- 見た目・触り心地は市販の抽選ルーレットと区別がつかないようにする（偽装が命）。

## ルーレット形式（確定）

**回転する円盤（ホイール）＋上部に固定の矢印**

- 円盤は項目数ぶんの扇形（セグメント）に分割。各セグメントにラベル（ドンペリ／ビール／…）。**初期は6セグメント・等分**。
- **セグメント幅は項目ごとに個別調整できる**（初期は等分だが、隠し設定で崩せる。詳細は後述「セグメント幅の個別調整」章）。
- 矢印は画面上部（12時方向）に固定。円盤が回転して減速し、矢印が指すセグメントが結果。
- セグメントの色は選択中のパレット（後述）から順に割り当てる（隣接色がかぶらないよう配色）。
- **描画は SVG**。セグメントは `<path>`、ラベルは `<text>`（各セグメントの中心角に沿って配置）。回転は円盤グループ全体を `transform: rotate()` で回す。テキストが鮮明で、画面幅に応じた可変にも強い。
- **ラベルは自動縮小**：セグメント幅に収まらない長さのときはフォントサイズを段階的に下げて1行に収める（省略も折返しもしない）。極端に長い場合の下限フォントは決めておく。

### 画面構成

```
┌─────────────────┐
│      ルーレット      │ ← タイトル（隠し設定を開くタップ領域）
│        ▼          │ ← 固定の矢印（12時／アームのON/OFFサインを兼ねる）
│    ╱───────╲      │
│   │ ドンペリ │     │
│   │ビール 円盤 ハイ│ │ ← 回転する円盤（SVG・6セグメント＋ラベル）
│   │  ワイン  │     │
│    ╲───────╱      │
│                   │
│    ┌─────────┐    │
│    │  START   │    │ ← START/STOP ボタン（トグル）
│    └─────────┘    │
└─────────────────┘
```

- タイトル文字（表示名）を上部に小さく置く（隠し設定を開くタップ領域に使う。後述）。
- 中央下に **START/STOP ボタン**（1つのボタンがトグル）。アイドル時は「START」、回転中は「STOP」表示。

## 演出の流れ

1. アイドル状態：円盤停止。ボタンは「START」。
2. START を押す → 円盤が一定の高速で回り続ける（減速はまだしない）。ボタンは「STOP」に変化。
3. STOP を押す（観客に押させてもよい）→ その瞬間から**自然に減速して当たりに着地**。
4. 停止：矢印が当たりセグメントを指す。少し余韻を持たせて結果を強調。ボタンは「START」に戻る。
5. もう一度アイドルへ（次の演技に備えてリロード不要）。

## フォースの仕組み（このアプリの技術的な肝）

電卓フォースの「帳尻の数字」と同じ発想。**止めた瞬間の現在角度から逆算して、狙った角度にちょうど着地させる**。

### 円盤の角度モデル（セグメント幅が不均一なことを前提にする）

- セグメント数 `N`。各セグメントは「開始角・終了角」を持つ（**幅は項目ごとに異なってよい**。合計は必ず360度）。等分表示のときは単純に `360/N` ずつだが、モデル自体は最初から可変幅を前提にする（等分は「幅がすべて同じ」という特殊ケースとして扱う）。
- 円盤の回転角 `_ang`（時計回りに増加、度）。矢印は常に12時固定。
- 当たりセグメント `_ti`（インデックス）を矢印下に持ってくる回転角を `targetAngle`（mod 360）とする。`targetAngle` は当たりセグメントの「開始角〜終了角」の範囲内でなければならない（後述ジッターもこの範囲内に収める）。

### 減速着地の計算（STOP を押した瞬間）

```
θ0        = 現在の回転角（mod 360）
targetAngle = 当たりセグメントを矢印下に置く角度（mod 360）＋ ジッター
delta     = ((targetAngle − θ0) mod 360)      // 0〜359：あと少しで着く分
extraTurns = 2〜4（現在速度が速いほど多く）    // 自然に見せる追加回転
θ1        = θ0 + extraTurns*360 + delta        // 最終到達角
duration  = 2.5〜4.0 秒（extraTurns に応じて）
```

- `θ0 → θ1` を **ease-out 系（easeOutQuart 等、終盤ほど強く減速）** でアニメーション。
- 終盤に「ジリジリと当たりに吸い込まれる」動きがルーレットらしさを生む。ここが自然かどうかで成否が決まる。

### ジッター（毎回ど真ん中に止めない）

- `targetAngle` に、当たりセグメントの実際の幅から算出した `±(そのセグメントの半幅 − マージン)` の範囲で微小なランダムオフセットを加える。
- 毎回セグメント中央ぴったりに止まると不自然・機械的なので、必ず当たり内に収まる範囲で揺らす。
- **セグメント幅が極端に狭い場合**（後述の「範囲の個別調整」で当たりを細くしたとき）：ジッターの余地がほぼ無くなる。その場合はジッター幅を0に近づけてよい（＝ほぼ中央固定）。矢印が寸分違わずその狭い枠を射抜く見え方自体が演出の売りになるため、無理に揺らす必要はない。

### 早すぎる停止のガード

- 回し始め直後に STOP されても、`extraTurns ≥ 2`・`duration ≥ 2.5秒` を最低保証。
- カクッと即停止させない（バレる）。

### ランダムモード（フォースOFF時）

- `targetAngle` を当たり固定ではなく**ランダムなセグメント**にするだけ。減速アニメは完全に同一。
- OFF時は本当にランダムに止まる → 事前に「本当にランダムでしょ？」と見せられる（弁明可能性）。
- **重要**：抽選は「円盤全周から一様に1点を選び、その点が含まれる範囲を当選とする」方式にする（＝当選確率は範囲の角度の広さに比例する）。「項目の数で均等に選ぶ」実装にしないこと。項目ごとに範囲の広さを変えられる本アプリでは、後者だと見た目の広さと実際の当選確率が食い違い、極端に狭くした項目でも五分五分で当たってしまう不具合になる（v0.5で発覚・修正済み）。

## 回転の開始・停止（確定）

**中央下の START/STOP ボタンで回す・止める（1つのボタンがトグル）**

| 状態 | ボタン表示 | 押すと |
|------|-----------|--------|
| アイドル | START | 回転開始（高速で回り続ける） |
| 回転中 | STOP | 減速開始 → 当たりに着地 |
| 減速中 | STOP（無効） | 反応しない（着地まで待つ） |

- ボタンがあることで誤操作が少なく、術者が主導権を握りやすい。
- 観客に「好きなところで止めて」と STOP を押させてもよい。
- **補助操作**：回転中は円盤本体をタップしても停止できる（端末ごと観客に渡したいとき用）。減速開始後のタップ・連打は無視。
- START/STOP と円盤タップ以外の画面領域は回転に反応させない（隠し操作と競合させないため）。

## 演出ON/OFF（アーム）とONサイン（確定：アーム式）

弁明可能性のため、既定はOFF（本当にランダム）。クライマックス直前に術者が「アーム」してフォース状態にする。ロック画面アプリの演出ONと同じ思想。

| 状態 | 挙動 |
|------|------|
| OFF（既定） | STOP で止めると**ランダム**に着地（デモ用・弁明用） |
| ON（アーム済み） | STOP で止めると必ず**当たり**に着地 |

- **アーム操作（v0.6で変更・計画段階）**: 上部の**矢印（▼）の長押し**でON/OFFトグル（アイドル時のみ有効）。従来の「素早く3回タップ」は誤操作・手間の両面で扱いにくいとの実機フィードバックにより、**1回の操作で完結する長押しに変更**する。長押しの判定時間・キャンセル条件（長押し中に指を離す／動かす等）は実装時に決める。中央は START/STOP ボタンが占めるため、引き続き矢印を仕込みポイントとする。
- **ONサイン**: ONの間だけ矢印の見た目がごくわずかに変化（色の濃さ or 影を1段階だけ）。術者だけが気づく程度に留める。矢印は常に視線の近くにあり確認しやすい。（変更なし）
- アーム状態は**セッションのみ保持**（リロードでOFFに戻る）。うっかりアームしたまま観客に渡ってもリロードで安全側に倒れる。（変更なし）
- 運用注意：観客が偶然、矢印を長押ししてしまうとON/OFFが切り替わる可能性がある（3回タップよりは偶発的に成立しにくいが、ゼロではない）。クライマックス前にONサインを目視確認すること（ロック画面アプリと同じ注意）。

## 設定パネル（項目と当たりの仕込み）※v0.6で入口を変更（計画段階）

**画面左上の「設定」ボタン（常時表示）をタップ** → 設定パネルを開く。

- 従来の「タイトル文字を3回連続タップ」は廃止する。設定パネルの中身自体がネタバレしない語（表示項目／基準項目／テーマ／範囲）で統一されているため、入口を隠す必要はないという判断（v0.6での方針転換）。普通の抽選ルーレットアプリとして見ても、設定ボタンがあること自体は不自然ではない。
- ボタンの見た目は歯車アイコン等、一般的な「設定」を示すもの。アイドル時のみ有効（回転中・減速中は無効。既存のガードと同じ考え方）。

| 項目 | UI上の文言（難読化） | 内容 |
|------|--------------------|------|
| 項目リスト | 「表示項目」 | ルーレットのラベルを追加・編集・削除（例：ドンペリ／ビール／ハイボール…） |
| 当たり | 「基準項目」 | 上のリストから1つを選択＝フォース先。UIでは「当たり」「フォース」と書かない |
| 項目数 | （リスト連動） | 初期6。リスト編集に追従（上限は要検討） |
| セグメント幅 | 「範囲」 | 各項目のスライダー等で幅の比率を調整。合計が360度になるよう自動調整 |
| 配色 | 「テーマ」 | 複数のプリセットパレットから選択（サムネイルで選ぶ）。選択で即プレビュー |
| 演出（v0.6で追加・計画段階） | 「演出」 | 基準項目以外の各項目について、着地時の演出を「中」「小」から選択（後述「出目ごとの簡易演出」参照）。基準項目は常に「大」固定でこの選択肢には出さない |
| 完了 | 「完了」 | 保存してパネルを閉じる |

### セグメント幅の個別調整（新機能）

普通の抽選ルーレットは等分割だが、このアプリは項目ごとに幅（＝出やすさの見た目）を変えられる。当たり（基準項目）の枠を極端に狭くしても必ずそこに止まる、という見え方がより不可能性の高い演出になる。

- 隠し設定の「範囲」で、各項目の幅を比率（重み）で調整する。例：6項目を均等でなく「ドンペリだけ極端に狭く、残り5項目で広く分け合う」ように設定できる。
- 幅の入力方式はスライダー等、直感的に増減できるUIとする（具体的なUI部品は実装時に決める）。
- 全項目の幅の合計は常に360度になるよう自動的に正規化する（術者が個別の度数を意識しなくてよい）。
- 最小幅の下限を設ける（0度に近すぎるとラベルが表示できず、当たりだと視覚的に不自然になるため）。下限値は実装時に決める。
- 円盤の描画（セグメントの色分け・ラベル配置）は等分・不均等いずれの場合も同じロジックで扱う（等分は「全項目の幅が同じ」という特殊ケース）。
- **フォース計算への影響**：目標角度・ジッターは、当たりセグメントの実際の幅（狭くても広くても）を基準に計算する（前述「円盤の角度モデル」「ジッター」参照）。狭い枠でも必ずその枠内に着地させる。

### 配色パレット（プリセット）

- 隠し設定の「テーマ」で複数のプリセットから選ぶ。各パレットは4〜8色程度の配列。
- セグメントには先頭から順に色を割り当て、隣接が同色にならないようにする（項目数がパレット色数を超える場合は循環＋隣接回避）。
- 具体的な色パターンは実装スプリントで数種類用意（例：カジノ系／パステル系／モノトーン系など）。名称はネタバレしない中立的な呼び方にする。

- 設定は localStorage に保存し、再起動後も保持（演技途中でアプリを閉じても消えない）。
- パネルの開閉は回転中は無効（回転中のタップは「止める」専用のため）。
- 「モード」「フォース」「当たり」等ネタバレ語はUIに出さず「基準項目」等で統一（難読化ルール準拠）。

## 豪華演出（大当たり演出）※6-1（演出本体）・6-2（写真登録）とも実装済み。出目ごとの中/小演出はv0.6で追加（計画段階）

観客から見て「たまたま止まった項目が、やたら盛大に祝われる」という体裁のご褒美演出。パチンコの大当たり・確変演出のように、笑えるくらい豪華にする。

### 出目ごとの演出3段階（v0.6・計画段階）

これまでは基準項目に着地したときの「大（FEVER）」演出のみで、それ以外の項目は静かな余韻だけで地味だった。**すべての出目に何らかの演出をつけ、基準項目以外にも「中」「小」の簡易演出を選べるようにする。**

| 段階 | 対象 | 内容 |
|------|------|------|
| 大 | 基準項目（固定・変更不可） | 既存の大当たり演出（FEVER）。全画面オーバーレイ＋紙吹雪＋ファンファーレ＋写真表示。実装済み・変更なし |
| 中 | 基準項目以外の項目のうち、設定で「中」を選んだもの | 矢印まわりに小規模な紙吹雪＋短い音。全画面オーバーレイは使わない |
| 小 | 基準項目以外の項目のうち、設定で「小」を選んだもの（既定） | 短い白フラッシュ（0.3秒程度）＋短い「ピロン」音のみ |

- **中／小の割り当ては項目ごとに設定パネルで決められる**（前述「設定パネル」章の「演出」項目）。基準項目は常に「大」固定でこの選択肢に出さない。
- 狙い：どの出目で止まっても「何かしらの演出があって楽しい」体験にしつつ、基準項目のときだけ明確に段違いの盛り上がりにすることで、大当たり演出の特別感を保つ。
- 中・小演出は既存の「余韻」表示に代わるものとして扱う（大当たり演出と同様、演出の間は次の回転操作を受け付けない。ただし中・小は短時間で自動的に終わってよい＝手動で閉じる操作は不要、という方向で実装時に検討する）。

### 発動条件（確定）

**着地した項目が基準項目（`_ti`）と一致したら、演出モード（`_am`）のON/OFFに関わらず常に発動する。**

- ON（フォース中）でも OFF（本当にランダム）でも、結果的に基準項目に止まったときは同じように盛大に祝う。
- 狙い：「フォース成立時だけ演出が出る」という規則性をなくし、「この項目は当たると豪華に祝われる特別な項目」という自然なカバーストーリーにする。OFF時のデモ中に偶然基準項目に止まっても、同じ演出が出ることで一貫性が保たれ、術者にとって都合が悪い規則性が生まれない。

### 表示内容

- 派手な祝福演出：「大当たり!!」「祝 確変突入!!」のような煽り文言、紙吹雪・フラッシュ等のアニメーション。笑えるくらいやりすぎな方向で良い。
- **文言の注意**：「GET」「獲得」など、その項目がもらえることを意味する表現は使わない（おかわりの**参考**として回すルーレットであり、景品と誤解されるため）。実装では「FEVER!!」を採用。
- **効果音**：Web Audio APIでファンファーレを合成して再生（外部音源ファイル不使用）。iOSの自動再生制限があるため、START/STOP押下時にAudioContextを準備しておき演出表示時に鳴らす。実装済み。
- **写真表示**：その項目に写真が登録されていれば、演出内で大きく表示する。未登録の項目は文字・演出のみで成立させる（写真必須にしない）。

### 写真機能（確定：項目ごとに任意登録）※実装済み

- 隠し設定パネルの「表示項目」一覧の各行に「写真＋／写真✓」ボタンを追加。各項目に任意で写真を1枚登録できる（基準項目に限らずどの項目にも登録可）。
- 取り込み時に自動加工：長辺900pxまで縮小し、JPEG品質0.8→0.6→0.4の順で900KB以下を狙って圧縮。1.4MB超は取り込み失敗とする（localStorage圧迫防止）。
- 保存先はlocalStorage（`_zk5`＝dataURL配列 `_ph`。並びは項目リスト `_sg` と一致）。項目の追加・削除時に同期し、読み込み時に長さを項目数へ整合。
- 登録済みボタンの再押下で「削除する／キャンセルして変更」を選べる。
- 大当たり演出内で、着地した基準項目に写真があれば金枠付きで大きく表示（未登録なら文字のみ）。

### 終了方法（確定：手動）

- **画面タップで閉じる**（自動タイマーでは閉じない）。術者が盛り上がりをコントロールしてから、任意のタイミングで閉じてアイドルに戻す。
- 演出を閉じるまでは次の回転操作（START等）は受け付けない。

### 表示タイミング（要検討）

- 着地直後、既存の「余韻」表示との関係を整理する必要がある（余韻を豪華演出に置き換えるのか、一瞬の余韻の後に豪華演出へ移行するのか）。実装時に決める。

### 難読化への配慮

- この演出自体は観客に見せるためのものであり、演出が出ること自体を隠す必要はない（結果が確定した後に見せる祝福なので、フォースの仕組みをバラすものではない）。
- ただし、コード上の変数名・関数名・localStorageキーは他機能と同様に難読化する方針を維持する（具体的な命名は実装時に決定）。

## ライセンス管理（確定：あり）

既存アプリ（stopwatch / calc）と同じ **ID・パスコード＋端末ロック方式**。詳細は `apps/stopwatch/docs/auth-spec.md` を踏襲する。ここでは差分のみ記載。

### 方式（実装済み）

- 起動時にログインゲートを表示。ID・パスコードで認証し、1IDにつき1端末のみ使用可（`deviceId` ロック）。
- Firestore（プロジェクト `nandeya-magic-app`）の **アプリ専用コレクション `roulette_licenses` / `roulette_deviceRequests`** を使う（calc/clock/cipher と同じくアプリ別コレクション方式＝「いつもどおり」）。ドキュメントID＝発行ID。
- 端末変更は `roulette_deviceRequests` に申請 → 管理画面で承認して `deviceId` を null に戻す運用。
- ログイン済み記録（`_rlicid`）と端末ID（`_rdevid`）は localStorage に保持。再訪時は Firestore で端末IDの有効性を確認してから通常画面へ（自動ログイン）。
- 難読化名：`_rag`（ゲート）/ `_rid` `_rpc`（入力）/ `_rlb`（ボタン）/ `_rdr`（端末変更）/ `_rtl`（ログイン）/ `_rtal`（自動ログイン）/ `_rdcr`（端末変更申請）/ `_grdi`（端末ID）/ window `__rdb` `__rfns` / イベント `_rfsready`。

### 管理画面（実装済み）

- **共有の `apps/stopwatch/admin.html` に「ルーレット」タブを追加**（新規ファイルは作らない。全アプリのライセンスを1画面で管理する既存方式に合わせる）。ID発行・一覧・削除・端末登録リセット・端末変更承認。要素IDは `ro` プレフィクス、コレクションは `roulette_licenses` / `roulette_deviceRequests`。

### Firestoreルール（要デプロイ）

- `apps/stopwatch/firestore.rules` に `roulette_licenses` / `roulette_deviceRequests` のルールを追加済み（clock/cipher と同じ `deviceId == null` パターン）。
- **このルールファイルは Firebase 側へ手動でデプロイする必要がある**（Firebaseコンソール or CLI）。未反映だと本番で末尾の全拒否ルールに当たり、ログイン時の `deviceId` 書き込みが拒否される。

### サブスク構想との整合（要相談）

- マジックアプリ群は月額サブスク（`members` コレクション＋Stripe＋Discord）への移行構想がある。ルーレットもそのプラットフォームに乗せる場合、認証は `licenses` 方式ではなく共通のサブスク会員判定に寄せる。
- **当面は既存の ID/パスコード方式で実装**し、プラットフォーム側が固まった段階で認証部分だけ差し替える方針とする（本体のフォース機能とは疎結合に保つ）。

## 難読化方針（最重要）

stopwatch / calc / lockscreen と同じ方針。コードを見ても「必ず当たりに止まる」仕掛けが分からないようにする。

- localStorage キー・関数名・変数名は意味のない名前。
- コメントは「補正」「基準」「表示調整」などの言葉。マジック用語（フォース・当たり・仕込み）を書かない。
- UIラベルも「表示項目」「基準項目」で統一。

## スマホ最適化

- `touchstart` / `touchend` でタップ応答（click遅延を避ける）。
- `requestAnimationFrame` で回転・減速をアニメーション。
- `user-select: none`、`-webkit-tap-highlight-color: transparent`、`touch-action: manipulation`。
- viewport: `width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no`。
- PWA対応: manifest.json、apple-touch-icon、ステータスバー黒。
- 縦向き前提。円盤は **SVG** で描画し、画面幅に応じて可変（`viewBox` で自動スケール、ラベルは常に鮮明）。
