# 時間巻き戻しマジックアプリ 製品仕様書 v0.1（試作）

**アプリ名（内部名）**: clock-force
**カバーストーリー**: シンプルな時計アプリ（ロック画面風）
**配置先**: `apps/clock/clock.html`（1ファイル完結）
**ホスティング**: `https://finaloji.github.io/nandeya-studio/apps/clock/clock.html`
**技術構成**: HTML/CSS/JavaScript（フレームワーク不使用）＋ Web Sensor API（DeviceMotion）

> SwiftUI版は Xcode（Mac専用）が必要なため、Windowsだけで開発・運用できるPWA版として作る。
> Core Motion → DeviceMotionEvent、振動 → navigator.vibrate で同等機能を実現する。

---

## マジックの全体像

演者のiPhoneだけを使う、時間が巻き戻って見えるマジック。

| 誰が | 何をするか |
|------|-----------|
| 演者 | この時計アプリを開き、1回タップしてセンサー許可を取る |
| 演者 | iPhoneを画面下向き（伏せ）でテーブルに置く |
| 演者 | 観客に「何分前に戻したいですか？」と聞く |
| 観客 | 「◯分前」と答える |
| 演者 | その分数に対応する方向へiPhoneを裏返して表向きにする |
| 画面 | 表向きになった瞬間、「現在時刻＋指定分数」を表示 |
| 画面 | 2秒後、1分ずつ高速で時刻が巻き戻る |
| 画面 | 本当の現在時刻まで戻ったら自動停止し、以降は通常の時計として動く |

---

## 演者の秘密入力（返す方向 → 分数）

伏せた状態から表向きに180°裏返す「回転の向き」で分数を指定する。

| 返す方向 | 初期値（分） |
|---------|------------|
| 手前（自分側へ倒す） | 3 |
| 右（右へ転がす） | 6 |
| 奥（向こう側へ倒す） | 9 |
| 左（左へ転がす） | 12 |

- この4つの分数は**設定画面で変更できる**（試作の後半スプリントで実装）。

### 技術的な前提（重要）

**4方向は「返している最中の回転方向（ジャイロ）」でしか区別できない。**
180°回転は、どちら回りに回しても最終的な向きは同じになるため、静止後の向きだけでは
「手前 / 奥」「右 / 左」を判別できない。よって伏せ→表向きの遷移中に
`DeviceMotionEvent.rotationRate` を積算し、支配的な回転軸とその符号から4方向を決定する。

- 手前 / 奥 … 端末の左右軸まわりの回転（beta方向）。符号で手前・奥を区別。
- 右 / 左 … 端末の上下軸まわりの回転（gamma方向）。符号で右・左を区別。

## 状態遷移

```
[許可待ち] --タップで許可取得--> [通常時計]
[通常時計] --伏せをN秒継続--> [待機モード]
[待機モード] --表向きに裏返す（方向判定）--> [プリセット表示]
[プリセット表示] --2秒経過--> [巻き戻し中]
[巻き戻し中] --現在時刻に到達--> [通常時計]
どの状態でも --右上を3回タップ--> [通常時計]（強制リセット）
```

| 状態 | 画面の見え方 |
|------|------------|
| 許可待ち | 時計は表示、画面に「タップして開始」的な控えめな案内 |
| 通常時計 | 現在時刻をリアルタイム表示（毎秒更新） |
| 待機モード | 見た目は通常時計のまま（伏せているので画面は見えない） |
| プリセット表示 | 現在時刻＋指定分数を静止表示 |
| 巻き戻し中 | 約0.5秒ごとに表示時刻が1分ずつ減っていく |

## タイミング・数値パラメータ（初期値）

| 項目 | 初期値 |
|------|--------|
| 伏せ検出の継続時間（待機モード突入まで） | 0.8秒 |
| 表向き確定からプリセット表示 → 巻き戻し開始まで | 2秒 |
| 巻き戻しの1ステップ間隔 | 0.5秒 |
| 巻き戻しの1ステップの減少量 | 1分 |
| 判定成功時の振動 | 約100ミリ秒（1回）※後半スプリント |

---

## 隠し操作

**画面右上を3回連続タップ** → 現在時刻へ強制リセット（どの状態からでも）。

- タップ間隔が空きすぎたらカウントをリセット（例: 各タップ間 1秒以内）。
- 判定ミスや途中でやり直したいときの保険。

---

## 表示フォーマット

- 時刻は `HH:MM`（24時間 or ロック画面風。試作では `HH:MM` を大きく中央表示）。
- 秒は通常時計時のみ更新（巻き戻し中は分単位で動く）。
- 日付の小さい表示は任意（試作では時刻のみでOK）。

---

## エッジケース

| ケース | 挙動 |
|--------|------|
| センサー非対応 / 許可拒否 | 通常時計としては動く。方向判定は無効（案内は控えめに） |
| 巻き戻し中にまた伏せる | 進行中の巻き戻しは中断せず、現在時刻到達で通常へ。誤操作は右上3回タップで復帰 |
| 指定分数が0や未確定 | プリセット表示に入らず通常時計のまま |
| 画面回転 | 縦向き固定（既存アプリと同様） |
| 再読み込み | 通常時計から再スタート（許可は再取得が必要な場合あり） |

## 難読化方針（マジックアプリ共通ルール）

- localStorageキー・関数名・変数名は意味のない略号にする（例: `_dcf`, `_rwd`, `_dmap`）。
- UIラベルは「表示設定」「補正」など、ネタがバレない言葉を使う。
- 設定画面の「4方向の分数」も「表示補正値」等の名目にする。
- 詳細な対応表は実装スプリントで確定させる（電卓アプリの難読化対応表に倣う）。
