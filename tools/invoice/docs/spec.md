# 見積書・請求書作成ツール v2 仕様書

> 作成日: 2026-07-08
> ステータス: ドラフト（レビュー中）

## 1. 目的

- 株式会社なんで屋の見積書・請求書を、社内メンバーがスマホから簡単に作成できるようにする
- 作成した書類・案件の状況（見積中〜入金済み）を社内で共有できるようにする
- 旧ツール（localStorage・単独端末）を全面的に作り直す

## 2. 利用者・利用環境

| 項目 | 内容 |
|------|------|
| 利用者 | 社内の複数メンバー |
| メイン端末 | スマホ（PCも利用可能なレスポンシブ対応） |
| 認証 | 共通パスコード方式（初回入力後は端末に記憶、以降入力不要） |
| 公開場所 | GitHub Pages `tools/invoice/`（旧ツールを上書き。URLは現行のまま） |

## 3. 技術構成

- 純粋な HTML / CSS / JavaScript（フレームワーク不使用）
- Firebase Firestore（Firebase Auth 不使用。共通パスコード方式）
- Firebase プロジェクトは本ツール用に**新規作成**（ストップウォッチとは分離）
- モバイルファースト
- PWA対応（manifest.json、apple-touch-icon）

### Firestore に保存するデータ

| データ | 内容 |
|--------|------|
| 案件（書類） | 宛先、明細、金額、ステータス、作成日など |
| 料金マスタ | マジシャン一覧と料金プラン、オプション、その他項目 |

### 端末内（localStorage）に保存するデータ

| データ | 内容 |
|--------|------|
| パスコード認証済みフラグ | 初回認証後の記憶 |
| 下書き | 入力途中の内容（自動保存） |

## 4. 機能一覧

### 4-1. 書類作成

- 見積書／請求書の切り替え
- 宛先入力（宛名・担当者名）
- 発行日（デフォルト: 今日）
- 支払期限（デフォルト: 来月末）— **請求書のみ記載。見積書には載せない**
- 明細行の追加・編集・削除（項目名・数量・単価）
- 交通費・値引き
- 消費税（10%）・合計の自動計算
- 備考欄（振込先のデフォルト文を自動セット）
- **書類番号欄は設けない**

#### 消費税の計算ルール

- 交通費は明細行として帳票に表示するが、**税込扱いで消費税の課税対象に含めない**
- 消費税（10%）は交通費を除いた小計に対して計算する
- 例（ひな型より）: 公演費 70,000 ＋ 交通費 3,000 → 小計 73,000、消費税 7,000（70,000 の10%）、合計 80,000

### 4-2. マジシャン別かんたん入力（旧ツールから継承）

- マジシャンを選択 → 料金プランのボタンをタップで明細に追加
- オプション・その他項目（照明代など）も同様にボタンで追加
- 料金データは Firestore の料金マスタから取得

### 4-3. 料金マスタ編集画面

- ツール内の設定画面から以下を編集できる（コード修正不要）
  - マジシャンの追加・削除・並び替え
  - 料金プラン（プラン名・金額）の追加・変更・削除
  - オプション、その他項目の編集

### 4-4. ステータス管理

案件ごとに以下のステータスを持つ。

```
見積中 → 受注 → 請求済 → 入金済み
             ↘ 失注
             ↘ キャンセル
```

- 一覧画面でステータス別に絞り込み・確認できる
- ステータスは手動で変更する

### 4-5. 見積書 → 請求書の変換

- 作成済みの見積書から、ワンタップで請求書を作成できる
- 宛先・明細・金額を引き継ぎ、発行日・支払期限は再設定
- 変換時にステータスを「請求済」へ進める（確認あり）

### 4-6. 案件一覧（旧「作成履歴」の発展形）

- Firestore に保存された案件を一覧表示（全メンバー共通）
- ステータス・宛名・金額・担当マジシャン・日付を表示
- 案件を開いて再編集・複製・削除ができる

### 4-9. 顧客管理

- 顧客マスタ（Firestoreの `customers` コレクション、1顧客1ドキュメント）を持つ
  - 項目：会社名／担当者名／電話番号／メール／住所／備考
- 「顧客管理」画面で一覧表示・追加・編集・削除ができる
- 案件作成時、宛先セクションで顧客マスタから選択すると宛名・担当者名が自動入力される（選択後も自由に編集可）
- 選択した顧客の `customerId` を案件データに保存し、顧客別売上集計に使う（自由入力のみの案件は `customerId` なしとして集計対象外〜「未登録顧客」扱い）

### 4-10. 売上ダッシュボード

- 集計対象：**請求書のみ**、ステータスが「請求済」または「入金済み」の案件（見積書・見積中の案件は含めない）
- 表示する軸：
  - 月別の売上合計（発行日の年月ごと）
  - マジシャン別の売上合計
  - ステータス別の内訳（請求済 / 入金済み）
  - 顧客別の売上合計
- 各軸は金額の大きい順に並べ、簡易な横棒グラフ（CSSのみ、外部ライブラリ不使用）で見やすくする

### 4-7. 下書き自動保存（旧ツールから継承）

- 入力途中の内容を端末内に自動保存
- ブラウザを閉じても次回開いたときに復元

### 4-8. PDF出力

- プレビュー表示 → ブラウザの印刷機能でPDF保存（A4縦）
- 帳票デザインは、現在使用中のひな型（受領済み）に似せて作る

#### 帳票レイアウト（ひな型ベース）

- 右上に発行日（`2026/01/17` のようなスラッシュ区切り）
- タイトル「見 積 書」／「請 求 書」は横長の枠線ボックスに中央寄せ
- 宛名は左上「〇〇〇〇 様」
- 発行者情報は右側（社名・橋本 大輝・住所・TEL・mail・登録番号。「代表」の肩書は付けない）
- 挨拶文「下記の通り御見積申し上げます。」（請求書は「御請求」）を小さな枠線ボックスで表示
- 金額ボックス「御見積金額　80,000 円」（請求書は「御請求金額」）を枠付きで目立たせる
- 明細表は「品名／数量／単価／金額」の4列（No.列なし）
  - 空行も含めて一定行数の枠を表示する（ひな型は14行程度）
  - 1行おきにグレーの縞模様
  - 金額は「¥」記号なしのカンマ区切り
- 表の下部は左に［内容詳細］（備考）、右に小計・消費税（10%）・合計を一体化して表示
- 請求書のみ追加で記載するもの:
  - 支払期限
  - 振込先（住信SBIネット銀行）

見積書と請求書は同一レイアウトで、タイトル・挨拶文・金額ラベル・支払期限・振込先の有無だけが異なる。
